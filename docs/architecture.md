# Архитектура MVP

## Выбранный стек

- Язык: Python
- Telegram framework: `aiogram 3`
- ORM: `SQLAlchemy 2`
- Основная БД: PostgreSQL
- Локальный fallback: SQLite
- Конфигурация: `.env` через `python-dotenv`

## Обоснование выбора

`aiogram` подходит для Telegram-ботов с командами, callback-кнопками, роутерами и FSM-сценариями. SQLAlchemy отделяет бизнес-логику от конкретной базы данных и позволяет использовать PostgreSQL для более устойчивой работы, сохраняя SQLite для локальной демонстрации.

PostgreSQL выбран как основная база для масштабирования, одновременной работы нескольких пользователей и дальнейшего деплоя. SQLite оставлен как простой вариант локального запуска.

## Компоненты

1. `src/main.py`
   Инициализирует конфигурацию, БД, бота, диспетчер, роутеры и фоновую задачу напоминаний.

2. `src/bot/config.py`
   Загружает `BOT_TOKEN` и `DATABASE_URL`, нормализует PostgreSQL URL для драйвера `psycopg`.

3. `src/bot/database/`
   Содержит SQLAlchemy-модели, базовый класс и создание сессий.

4. `src/bot/handlers/`
   Обрабатывает команды `/start`, `/menu`, `/help`, ручной ввод слов и callback-навигацию.

5. `src/bot/keyboards/`
   Формирует inline-кнопки главного меню, обучения, тренировок, словаря и настроек.

6. `src/bot/services/`
   Содержит бизнес-логику пользователей, обучения, словаря, тестов, прогресса, настроек и напоминаний.

7. `src/bot/states.py`
   Описывает FSM-состояния для ручного добавления слова.

## Логика уровней

Темы распределены по языкам и уровням A1, A2, B1 и B2. Поддерживаются английский, испанский и итальянский языки. Пользователь выбирает язык при первом запуске и может менять его в настройках.

Доступ к уровням работает накопительно:

- A1: темы A1;
- A2: темы A1 и A2;
- B1: темы A1, A2 и B1;
- B2: все темы.

Такой подход позволяет пользователю высокого уровня повторять базовые слова, а не терять к ним доступ.

Прогресс, слабые слова и личный словарь считаются отдельно для текущего языка. При смене языка старые результаты не удаляются, потому что они связаны с карточками конкретного языкового набора.

## Хранение данных

В БД хранятся:

- пользователи Telegram;
- настройки пользователя;
- выбранный язык обучения;
- учебные темы;
- карточки слов;
- личный словарь;
- прогресс по конкретным словам;
- попытки тестов;
- дата последней отправки напоминания.

Пользовательские слова отделены от системных учебных тем и не показываются другим пользователям.

## Напоминания

Фоновая задача раз в минуту проверяет пользователей с включенными напоминаниями и отправляет сообщение в выбранное время. Чтобы не отправлять одно и то же напоминание несколько раз в день, хранится дата последней отправки.

## Текущие ограничения

- бот работает через polling;
- напоминания работают только пока процесс бота запущен;
- для постоянной работы нужен серверный деплой;
- миграции реализованы на уровне простых ALTER TABLE;
- для промышленной версии лучше добавить Alembic, Docker и webhook.
